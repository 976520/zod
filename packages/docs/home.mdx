## Introduction

Zod is a TypeScript-first schema declaration and validation library. I'm using the term "schema" to broadly refer to any data type, from a simple `string` to a complex nested object.

```ts
import { z } from "zod";

const User = z.interface({
  username: z.string(),
});
```

We can use Zod schemas to *parse data*. The result of this operation is properly typed and fully validated. If validation fails, a `ZodError` will be thrown.

```ts
User.parse({ username: "ludwig" });   // => { username: "ludwig" }
User.parse({ evil: true });           // => throws ZodError
```

Zod *infers* a static type from your schemas. This way, you never need to redundantly define TypeScript types and their associated validation logic. Instead, you declare your Zod schema _once_ and can extract the type from it.

```ts
type User = z.infer<typeof User>;
// { username: string };

const myUser: User = { username: "ludwig" }
```

## Features

- Zero dependencies
- Works in Node.js and all modern browsers
- Tiny: 8kb minified + zipped
- Immutable API: all methods return a new instance
- Concise interface
- Works with TypeScript and plain JS
- Extensive ecosystem

## Installation

### Requirements

You must enable `strict` mode in your `tsconfig.json`. This is a best practice for all TypeScript projects.

  
  ```ts
  // tsconfig.json
  {
    // ...
    "compilerOptions": {
      // ...
      "strict": true
    }
  }
  ```

<Info>Zod is tested against *TypeScript v5.5* and later. Older versions may work but are not officially supported.</Info>


### From `npm`

```sh
npm install zod       # npm
deno add npm:zod      # deno
yarn add zod          # yarn
bun add zod           # bun
pnpm add zod          # pnpm
```

Zod also publishes a canary version on every commit. To install the canary:

```sh
npm install zod@canary       # npm
deno add npm:zod@canary      # deno
yarn add zod@canary          # yarn
bun add zod@canary           # bun
pnpm add zod@canary          # pnpm
```

> The rest of this README assumes you are using npm and importing directly from the `"zod"` package.
